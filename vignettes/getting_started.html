Getting Started with spectreasy

spectreasy: spectral unmixing on example FCS files

This vignette runs a complete spectreasy workflow on example FCS files stored as a GitHub release asset. spectreasy_example_data() downloads the archive matching the installed package version on first use, unzips it automatically, and returns local paths to:

  • single-color controls in scc/
  • one experimental sample in sample/sample.fcs

All code is evaluated during vignette builds.

suppressPackageStartupMessages(library(spectreasy))

Installation

For the released Bioconductor version, install with:

BiocManager::install("spectreasy")

If you need the current development version from GitHub, install with:

remotes::install_github("pkheisig/spectreasy")

remotes is used here rather than devtools because only GitHub installation is needed.

1. Download the example data and prepare a temporary project

We first download the example-data archive from the GitHub release matching the installed package version. spectreasy_example_data() caches the zip file and unzips it automatically. We then copy the extracted sample/ and scc/ folders into a temporary working directory so that the vignette writes outputs outside the cache.

example_paths <- spectreasy_example_data()

project_dir <- file.path(tempdir(), "spectreasy_vignette_project")
if (dir.exists(project_dir)) {
  unlink(project_dir, recursive = TRUE, force = TRUE)
}
dir.create(project_dir, recursive = TRUE)
dir.create(file.path(project_dir, "scc"))
dir.create(file.path(project_dir, "sample"))

file.copy(
  list.files(example_paths$scc_dir, full.names = TRUE, pattern = "\\.[Ff][Cc][Ss]$"),
  file.path(project_dir, "scc")
)
#> [1] TRUE TRUE TRUE TRUE TRUE TRUE TRUE
file.copy(
  list.files(example_paths$sample_dir, full.names = TRUE, pattern = "\\.[Ff][Cc][Ss]$"),
  file.path(project_dir, "sample")
)

4. Build the control-stage outputs

autounmix_controls() builds the reference matrix, saves control-stage plots, and unmixes the controls.

ctrl <- autounmix_controls(
  scc_dir = file.path(project_dir, "scc"),
  control_df = control_df,
  cytometer = "Aurora",
  output_dir = file.path(project_dir, "spectreasy_outputs", "autounmix_controls"),
  build_qc_plots = FALSE,
  seed = 1
)

dim(ctrl$M)

The control-stage matrix is labeled by fluorophore, which is useful for control QC. For downstream sample interpretation, we relabel the matrix rows to marker names using fcs_mapping.csv.

marker_map <- stats::setNames(control_df$marker, control_df$fluorophore)
reference_matrix <- ctrl$M
rownames(reference_matrix) <- unname(marker_map[rownames(reference_matrix)])

We can visualize the non-autofluorescence spectra directly from the evaluated matrix.

reference_matrix_no_af <- reference_matrix[rownames(reference_matrix) != "Autofluorescence", , drop = FALSE]
plot_spectra(reference_matrix_no_af, output_file = NULL)

plot_ssm(calculate_ssm(reference_matrix_no_af), output_file = NULL)

5. Unmix the example sample

Now we unmix sample/sample.fcs using the marker-labeled reference matrix.

unmixed <- unmix_samples(
  sample_dir = file.path(project_dir, "sample"),
  M = reference_matrix,
  method = "WLS",
  output_dir = file.path(project_dir, "spectreasy_outputs", "unmix_samples"),
  write_fcs = TRUE
)

sample_results <- unmixed$sample$data
